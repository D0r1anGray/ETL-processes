# Итоговое задание по модулю: ETL-процессы (Airflow, MongoDB, PostgreSQL)

## Описание проекта

Данный проект представляет собой реализацию полноценного ETL-пайплайна для сбора, трансформации и анализа данных. Система генерирует синтетические данные о сессиях пользователей и обращениях в поддержку в NoSQL базу данных, после чего реплицирует их в реляционное хранилище для построения аналитических витрин.

## Технологический стек

* **Apache Airflow**: Оркестрация пайплайнов.


* **MongoDB**: Исходная нереляционная база данных (хранение сессий и тикетов).


* **PostgreSQL**: Реляционная база данных (хранение очищенных данных и витрин).


* **Docker & Docker Compose**: Контейнеризация всей инфраструктуры.
* **Python (Pandas, SQLAlchemy, PyMongo)**: Извлечение и трансформация данных.

## Структура проекта

* `docker-compose.yaml` — описание инфраструктуры (Airflow, Postgres, Mongo).


* `dags/etl_main_dag.py` — основной DAG, реализующий ETL-процесс.


* `dags/fill_mongo.py` — скрипт для генерации первичных данных в MongoDB.



## Реализованные этапы

### 1. Генерация данных

Реализовано наполнение MongoDB данными о сессиях пользователей (`UserSessions`) и обращениях в поддержку (`SupportTickets`). Данные включают идентификаторы, временные метки и детали событий.

### 2. Репликация и трансформация (ETL)

Пайплайн в Airflow выполняет следующие шаги:

* **Extract**: Извлечение данных из MongoDB.
* **Transform**: Очистка данных и расчет бизнес-метрик (например, вычисление длительности сессии в секундах как разницы между `end_time` и `start_time`).


* **Load**: Загрузка трансформированных данных в промежуточные (staging) таблицы PostgreSQL.

### 3. Аналитические витрины

В PostgreSQL сформированы две витрины:

1. **Витрина активности пользователей (`mart_user_activity`)**: Анализ количества сессий и среднего времени взаимодействия с сервисом.


2. **Витрина эффективности поддержки (`mart_support_efficiency`)**: Статистика тикетов по статусам и типам проблем для оценки работы службы поддержки.



## Как запустить

1. Склонируйте репозиторий.
2. Запустите контейнеры:
```bash
docker-compose up -d

```


3. Зайдите в интерфейс Airflow по адресу `localhost:8080` (логин/пароль: `admin`/`admin`).
4. Активируйте и запустите DAG `final_etl_pipeline`.
